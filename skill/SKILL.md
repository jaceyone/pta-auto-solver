# PTA 自动解题与提交

自动化完成 PTA 平台的「登录 → 抓题 → 解答 → 提交」全流程。核心难点在于 PTA 使用
CodeMirror 6 编辑器，普通粘贴常常失效，因此本技能内置了内部 API 注入方案。

## 交互原则：默认自主，仅在必要处停下

这个技能的目标是尽量减少来回打断。请默认**自主连续执行**整条流程，为可逆的步骤
选择合理默认值并继续，不要每一步都征求同意。之所以这样设计，是因为大多数步骤
（选内核、翻页、点选答案、进入下一题）即便判断有偏差也很容易纠正，频繁确认反而
拖慢用户、消耗耐心。

只在以下**三个真正高风险或不可逆**的节点必须停下等待用户：

1. **需要凭据时**——登录 PTA 必须由用户亲自输入账号密码。
2. **确定抓取范围时**——第一次需要知道用户想做「哪一类 / 哪些」题目，避免误抓一大批无关题目。
3. **遇到无法自动解答的题目时**——不要伪造答案，交回给用户处理。

其余情况请自行决策并推进；如果某个默认判断可能有争议，采用「先执行、并在回复里简短说明
我做了什么、如何撤销」的方式，而不是提前拦一道确认。

## 前置：检测并安装 Playwright MCP

在打开浏览器前，先确保 Playwright MCP 已就绪。按以下顺序执行，能自动完成的不要问用户：

### 1. 检测是否已安装

检查当前环境的 MCP 配置中是否已存在 Playwright MCP（例如查看 MCP 客户端配置文件里
是否有 `@playwright/mcp` 或 `playwright` 相关条目，或直接尝试列出其提供的工具）。
若已存在且可用，跳过安装，直接进入「启动」。

### 2. 安装 MCP 服务

若未检测到，安装官方 Playwright MCP。典型方式是在 MCP 客户端配置中加入服务条目：

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"]
    }
  }
}
```

或在终端预拉取：`npx -y @playwright/mcp@latest --help` 以确认可执行。
若环境缺少 Node/npm，先提示用户安装 Node.js（这属于环境缺失，需告知用户）。

### 3. 确认浏览器内核

Playwright 需要一个浏览器内核。**默认选用 chromium**（覆盖面最广），无需为此打断用户；
仅当用户主动表达过偏好，或 chromium 安装失败时，才切换到其他内核。三种内核对应关系：

- **chromium** —— Chrome / Edge / 360 / QQ 等所有 Chromium 系（默认首选）
- **firefox** —— Mozilla Firefox（Gecko 内核）
- **webkit** —— Mac / iOS Safari（WebKit 内核）

首次运行需下载内核，可执行 `npx playwright install chromium`
（或 `firefox` / `webkit`）。下载可能较慢，属正常现象。

### 4. 启动与验证

通过 Playwright MCP 启动浏览器，并做一次轻量验证（如打开一个空白页或直接进入下一步的
PTA 首页），确认工具链通畅。若启动失败，读取报错并自行尝试常见修复（重装内核、
换内核、检查网络），仅在多次失败后才把具体错误汇报给用户。

## 第一步：登录与进入题目列表

1. 打开 https://pintia.cn/home
2. **停下来让用户输入账号密码**（这是必须的确认点之一；绝不猜测或存储凭据）
3. 登录成功后自动跳转到题目列表界面，无需再确认

## 第二步：分析页面结构

进入后自动通过 DOM 分析题目结构，识别题目按钮、选项、输入框、代码区等。

- PTA 的代码编辑器是 **CodeMirror 6**，容器 class 为 `.cm-editor`
- 具体选择器清单见 `references/selectors.md`

## 第三步：抓取题目

1. 自动查看平台上有哪些题目分类
2. **停下来让用户选择要抓取哪一类题目**（第二个必须确认点，防止误抓）
3. 用户确定后，派生子代理（Agent）自动翻页抓取该类全部题目，翻页过程无需逐页确认

## 第四步：解答并自动提交

确定范围后连续解题，不要每题都问。

### 选择题 / 判断题（可直接点选）

对判断题、单选、多选，直接点击正确选项，可用脚本批量点击所有正确选项，
再点击「保存」按钮提交。

### 编程题（代码输入）

自动判断当前编辑器是否支持粘贴：

- **能粘贴**：直接粘贴代码后提交
- **不能粘贴**：使用 CodeMirror 6 内部 API 注入（`content.cmTile.view.dispatch()`），
  正确处理换行与缩进。脚本见 `scripts/inject_code.js`、原理见 `references/codemirror.md`

### 带图片的题目

- **具备图片识别能力**：直接识别并解答
- **无法识别**：把图片下载到工作目录，然后**停下来告知用户无法自动解答**
  （第三个必须确认点），请用户用文字提供题目内容，再由你填充解答

## 会话与状态管理

- 批量做题会让上下文变长。适时**主动提醒**用户：若上下文过长，建议执行 `/clear`
  （这是提醒，不是征求同意）
- 每次开始时自行判断界面状态；只有当无法确定用户意图（开始新任务 vs 接续上次
  `/clear` 过的任务）时，才简短询问一次

## 安全与边界

- 绝不存储或回显用户的 PTA 账号密码
- 遇到无法解答的题目如实告知，不伪造答案
